# 论文创新点生成工作流

本 Skill 用于在**已有论文和资料已经初步收集完毕**的前提下，系统化分析现有创新点、缺陷与改进空间，并结合外部知识库（如网络搜索结果、开源代码、实践经验等），生成在某一**指定领域**中的新颖且可实施的创新点方案。  
目标是帮助你从「已有工作总结 → 空白/不足识别 → 新方案设计 → 可复现性评估 → 形成文档」这一完整链路，高效产出高质量的创新点列表与对应说明。

## 何时使用

**在以下情况使用此 skill：**
- 已经完成某一领域相关论文的**调研与归纳**，需要在此基础上继续挖掘新的创新点
- 希望在论文选题、项目立项、开题报告中，系统整理和生成**多组候选创新方案**
- 需要对现有论文的创新点进行**对比、复用与组合**，并结合开源实现评估可行性与可复现性
- 想将零散的阅读笔记、网络检索结果、经验知识整合为**结构化的创新点设计文档**

**改用其他方案当：**
- 仍处在「完全不了解该领域」阶段，尚未完成基础文献调研（此时更适合使用“文献调研/综述类”Skill）
- 仅需要对**单篇论文**做简单总结或翻译，而非在多篇工作基础上构造新的方案
- 需求只是做**实现复现/代码跑通**，而不是提出新的研究点或方法改进

## 快速开始

### 创新点生成整体工作流

1. **理解任务需求（领域 & 目标）**  
   - 明确目标领域（如「大模型推理加速」「3DGS场景重建」「自动驾驶场景重建」等）  
   - 明确输出形式：是顶层创新点列表，还是具体章节创新（如方法、实验、系统架构等）

2. **建立基础知识库**  
   - 通过网络搜索、survey、经典论文，梳理该领域的**基本概念、任务形式、主流范式**  
   - 将关键信息整理为要点，方便后续进行对照和创新

3. **阅读与梳理现有论文资料**  
   - 对已有论文进行结构化记录：问题设定、方法要点、创新点、实验设置、局限性等  
   - 形成一份「现有工作表格 / 清单」，作为后续组合创新的素材库

4. **归纳总结现有创新点、缺陷与潜在改进方向**  
   - 提炼每篇论文的关键创新点  
   - 标注其**不足之处**（适用范围、开销、复现难度、实验设置局限等）  
   - 提出初步的「潜在改进问题清单」

5. **生成新的创新点**  
   - 基于已有创新点的**组合、变体、约束放宽/增强、模块替换**等方式构造新方案  
   - 引入外部知识（产业需求、最新开源项目、工程瓶颈、其他领域方法）丰富思路  
   - 输出若干候选创新点，并附简要原理 & 预期优点

6. **形成结构化文档**  
   - 将最终筛选后的创新点整理为文档：  
     - 每个创新点的背景、动机、核心思想、实现思路、可复现性评估  
   - 为后续论文写作、项目实施或开题材料直接复用

### 基本使用流程

当你需要在某一**指定领域**上生成新的创新点时，可以按以下步骤使用本 Skill：

1. **归纳现有创新点（聚焦指定领域）**
   - 现有创新点有哪些？
     - 查询/root/git/docs/papers/_index.md中已有总结的指定领域论文
     - 从/root/git/docs/papers/summaries中查询论文详细总结，提取创新点
   - 都解决了哪些具体问题？  
     - 问题设定/任务场景  
     - 数据规模、模型规模、硬件环境等前提条件
   - 有哪些明显或隐含的不足？  
     - 性能瓶颈、泛化能力、场景受限、实现复杂度高、依赖特殊硬件/数据等
   - 可复现性如何？  
     - 是否有开源代码或公开实现细节  
     - 训练/推理资源要求是否现实可行  
     - 是否存在实现细节缺失或论文-代码不一致的情况

2. **进一步理解与扩展视野**
   - 通过网络搜索补充该领域的：  
     - 最新进展、竞品方案、工业界实践  
     - 相关任务的不同问题设定/评测指标
   - 对具有借鉴意义的论文或方案：  
     - 使用 `read-arxiv-paper` 的 **Pass 3** 做深入阅读与总结  
     - 提炼可迁移的思想、架构模式或训练策略

3. **尝试生成新的创新点**
   - **组合现有创新点**：  
     - 将不同论文中的优势部分合理拼接或改造  
     - 将在其他任务中成功的机制迁移到当前任务  
   - **过滤不可落地部分**：  
     - 剔除严重依赖未开源代码、私有数据、极端算力的设计  
     - 破解「看起来很新但无法复现」的伪创新
   - **补全细节，提高合理性与完整度**：  
     - 明确输入输出形式、训练/推理流程、关键模块接口  
     - 给出可能的实验设置与评测指标，避免停留在概念层面  
     - 预先考虑潜在风险与局限，为后续改进预留空间

4. **创新点评估与反思**
   - 与已有论文进行正面对比：  
     - 具体超越了哪些已有方案？（效率、性能、可扩展性、工程成本等）  
     - 是否只是已知方法的简单组合，还是有实质性新观点/新机制？
   - 评估可复现性与可应用性：  
     - 能否基于公开数据集与开源框架在常规 GPU 条件下实现？  
     - 是否有可直接借鉴的开源代码或实现参考？  
     - 如果需要自研模块，其实现复杂度与风险是否可控？
   - 对每个候选创新点给出**简短结论**：  
     - 「推荐重点推进」/「可作为备选」/「概念有趣但落地风险较大」

## 核心功能

### 功能 1：现有创新点系统化梳理

- 将多篇论文的创新点整理为统一结构（问题、方法、优势、局限、资源要求）  
- 自动提示容易被忽略的维度（如评测设置偏差、隐含假设、实现难度）

### 功能 2：结合外部知识的创新点生成

- 基于网络搜索、开源项目和领域背景知识，丰富创新空间  
- 鼓励跨任务、跨模态、跨范式的思想迁移与组合，避免局限在单一论文族谱内

### 功能 3：面向落地与复现性的评估

- 从开源代码可用性、复现实验所需资源、实现复杂度等维度评估候选创新  
- 对「理论上很新但工程风险大」的方案给出明确提醒与优化建议

## 最佳实践与提示

- **尽量先完成一轮「宽覆盖」的文献调研**，再进入创新点生成阶段，避免视野过窄  
- 对每个创新点都给出**至少 1 个潜在对比基线**，方便后续实验与写作  
- 优先保留那些**有明确实现路径、可在现有资源下落地**的创新点  
- 在文档中保留「被否决的创新点及原因」，为未来新条件/新需求下的再利用留下线索
